# Origin - AI-First Design-to-Code Platform

## 项目愿景

Origin 是一个类似 Penpot 的设计工具，但核心理念完全不同——**为 AI 辅助的人机协作而生**。

我们的目标是让设计师、开发者通过 AI 的协助，从设计到代码的全流程自动化，最终生成可直接运行的前端应用。

## 核心理念

### 工作流程

```
人类设计师/开发者
    ↓
AI 辅助设计
    ↓
UI 设计稿 + UX 设计 + 前端建模 + 前端系统外部能力
    ↓
DSL (领域特定语言)
    ↓
代码生成器
    ↓
可运行的应用代码
```

### 核心特性

1. **AI-First 设计体验**
   - 自然语言交互生成界面
   - AI 辅助布局调整
   - 智能组件推荐

2. **统一前端框架支持**
   - 优先支持 Flutter（跨平台移动/桌面/Web）
   - 设计一次，生成多平台代码

3. **OpenAPI 后端对接**
   - 导入 OpenAPI JSON 规范
   - 自动生成 API 调用代码
   - 类型安全的网络请求

4. **可视化建模**
   - 状态建模
   - 数据流建模
   - 交互逻辑建模

## 项目目标

### 短期目标 (MVP)
- [ ] 基础画布和图形绘制能力
- [ ] 简单的 DSL 设计
- [ ] Flutter 代码生成器
- [ ] OpenAPI 导入功能

### 中期目标
- [ ] AI 辅助设计功能
- [ ] 完整的状态建模系统
- [ ] 组件库管理
- [ ] 实时预览

### 长期目标
- [ ] 多框架支持 (React Native, SwiftUI)
- [ ] 协作编辑
- [ ] 云端部署
- [ ] 插件系统

## 与 Penpot 的区别

| 特性 | Penpot | Origin |
|------|--------|--------|
| 主要目标 | 设计协作工具 | AI 辅助设计到代码 |
| 代码生成 | 无 | 核心功能 |
| AI 集成 | 无 | 深度集成 |
| 后端对接 | 无 | OpenAPI 原生支持 |
| 输出 | 设计稿 | 可运行应用 |

## 项目命名

**Origin** 寓意着"起源"——这是应用诞生的起点，从想法到设计，再到代码，一切始于此。
